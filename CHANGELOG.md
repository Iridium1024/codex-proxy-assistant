# Changelog

## 0.1.3 - 2026-07-18

- 将 GUI、CLI 和独立 CMD 收敛到同一 PowerShell 核心，CMD 改为自动生成。
- 区分 Codex Desktop 与 CLI，优先 Desktop，并为每个候选增加独立超时和进程树终止。
- 将代理检测升级为端口状态与真实 HTTPS 链路分层验证，增加 407、超时、TLS 和连接重置分类。
- 临时 CLI 模式保留原有 `NO_PROXY` 并补齐本地地址绕过。
- 增加用户主动确认后才执行的最小真实 Codex 连接验证。
- 完善中文/空格路径、多安装形态和异常网络环境下的兼容性。
- 简化新手主流程、状态提示和 README。
- 正式提供 GUI、PowerShell 文字窗口和单文件 CMD 三种分发形态，并将文字窗口版作为多数用户的推荐下载。

## 0.1.1 - 2026-07-17

- 优化公开源码与发布包目录结构。
- 修正发布压缩包目录结构，解压后第一层即可看到 `CodexProxyAssistant.exe`。
- 发布打包流程增加目录结构自动校验，避免再次生成带多余外层文件夹的压缩包。

## 0.1.0 - 2026-07-17

- 提供面向 Windows 的轻量 GUI，可检测 Codex 路径、系统代理和配置状态。
- 支持预览、配置快照、持久修复、快照恢复和临时 CLI 代理试运行。
- 保留兼容 Windows PowerShell 5.1 的独立 `codex-vpn-repair.cmd`。
- 统一紧凑界面中的控件高度、文字对齐、下拉列表行高与最小窗口布局。
- 增加安全构建、哈希清单和 GitHub 发布包生成流程。
